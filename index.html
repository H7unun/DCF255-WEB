<INTRODUCTION

 Cyber security is the art of protecting networks, devices and data from unauthorized access or criminal use and the practice of ensuring confidentiality, integrity, and availability of information. The main goal of this study is to inform your personnel about the harmful security flaws so that future attacks won't be possible. This report will also give you the best security precautions to take to prevent future cybercrime against your business. 

The mission-critical assets you are trying to secure should be the focal point of the seven cybersecurity levels. 

"Companies have a tendency to value everything as imperative just in case if they don't know what data they have or how to organise it. The secret to success when selecting how to protect your data and apps is to determine what is mission-critical, or what your firm cannot function without." 

 

  1. <Mission Critical Assets – This is the data you need to protect. Devices, programmes, and databases are mission-critical assets because they form the backbone of your company. The business could not function without them. Therefore, it makes logical that your security plan is composed of several levels, each of which serves a different purpose in defending your mission-critical assets. In this, the business works with about 80 body shops and accepts orders through a small, hosted website. Additionally, the business was the target of a ransomware attack that encrypted all their backend databases. Therefore, we must use security measures to protect the website. 

 

  1. <Data Security - Every organization depends on its data to function. It helps with decision-making, problem-solving, increasing the effectiveness and efficiency of operations, enhancing customer service, informing marketing initiatives, lowering risks, boosting productivity, fostering teamwork, and ultimately contributing to rising revenue and profit. 

Data is frequently described as a company's "crown jewels," and as such, it requires considerable consideration to ensure its security. Data security is achieved when security measures are used to safeguard both the transfer and storage of data. To prevent data loss, a backup security solution must be in place. This calls for the usage of encryption and archiving. 

All firms place a high priority on data security since a data breach could have disastrous effects. 

An organisation must first be aware of the data it has before it can secure it. A data inventory, which is a list of all the data an organisation has created, consumed, and stored, is essential in this situation. Data discovery, or finding out what and where the data is, is the first step in the process. The process of classifying data entails giving it names so that it will be simpler to handle, store, and secure. The following are the four standard data classification categories: public information, confidential information, sensitive information and personal information. 

There are different types of data security technologies such as firewalls, Authentication/authorization, Encryption, Data masking, Hardware – based security, data backup and resilience. 

We take care of the organization's data security after the mission-critical assets. By using this approach, we safeguard the privacy, accuracy, and accessibility of an organization's data. Data must be accessible to authorised individuals, maintained in confidence, and have integrity to guarantee accuracy and dependability. 

 

  1. <Application layer: Application security seeks to defend data and software application code from online dangers. Application security may and should be used throughout the whole development process, including design, development, and deployment. In order to increase application security across the software development lifecycle (SDLC), consider the following strategies: 

Introduce security tools and standards while designing and developing applications. Consider including vulnerability screening in the early stages of development. 

Put security policies and methods in place to safeguard apps in production settings. For instance, carry out ongoing security testing. 

For applications that require mission-critical access or contain sensitive data, use strong authentication. Use security tools like intrusion detection systems, web application firewalls, and firewalls (IPS). 

Software that is available over the Internet and is hosted on a web server is referred to as a web application. In a web browser, the client is used. Applications are compelled to accept connections from users over insecure networks by nature. They become vulnerable to a number of threats as a result. A primary priority for any cyber security programme is to protect online apps, which are frequently business-critical and contain sensitive customer data, making them an attractive target for attackers. 

The advent of HTTPS, which establishes an encrypted communication channel and defends against man in the middle (MitM) attacks, is one example of how the Internet's evolution has addressed web application weaknesses. We perform the application security following the data security. And the WAF will be used for this. HTTP traffic that travels between a web application and the Internet is watched over and filtered by a WAF. While WAF technology cannot protect against every danger, it can be used in conjunction with a variety of security solutions to provide a comprehensive protection against several attack vectors. 

Web applications are protected by WAF, which acts as a protocol layer seven defence in the open systems interconnection (OSI) paradigm, from threats such cross-site scripting (XSS), cross-site forgery, SQL injection, and file inclusion. 

A WAF functions like a reverse proxy to protect the server from exposure, in contrast to a proxy server, which hides the identities of client machines through an intermediary.  

RASP can be added to our website as well. Runtime application traffic and user behaviour analysis is possible with RASP technology. Through gaining visibility into application source code and examining vulnerabilities and weaknesses, it seeks to assist in the detection and prevention of cyber threats. 

To offer proactive security, RASP technologies can spot security flaws that have already been exploited, stop these sessions, and send out alerts.  

 

  1. <End point layers: The cybersecurity strategy for protecting endpoints, such as desktops, laptops, and mobile devices, against malicious activities is known as endpoint security, or endpoint protection. Any device that connects to the business network from outside its firewall is considered an endpoint. Because every remote endpoint might serve as an attack's point of entry and because of the massive shift to remote work that the pandemic has caused, endpoints are becoming increasingly important. A Gallup survey found that 51% of US employees were still working remotely in April 2021, up from a majority in 2020. Endpoint hazards and the sensitive data they contain are an ongoing problem. Businesses of all sizes are prime candidates for cyberattacks because to the endpoint landscape's continual change. Endpoint security refers to the centralised security management tools that businesses use to safeguard their endpoints, such as servers, workstations, mobile devices, and workloads, from online threats. Endpoint protection programmes search for indicators of suspicious or malicious activities in files, processes, and system activity. Endpoint protection programmes provide a centralised management interface through which administrators may access their company's network for the purposes of monitoring, securing, looking into, and responding to problems. By using an on-premises, hybrid, or cloud strategy, this is achieved. We will employ the on-premises security posture in this layer. Since the management console communicates with the endpoints via an agent to provide security, the data centre serves as the hub for this process. Due to the fact that administrators often can only manage endpoints inside their perimeter, the hub and spoke approach might result in security silos 

 

  1.  <Network Layer: A lot of shared networks, like the Internet, have seen a rise in the adoption of network layer security controls for communication security since they can protect several applications simultaneously without altering them. Network security measures guard a company's network and stop unauthorised access to it. The popular framework developed for ensuring security at network layer is Internet Protocol Security (IPsec). 

<Features of IPsec: 

  • The IPsec protocol was not created to be exclusively compatible with TCP. Any protocol above IP, such as ICMP, OSPF, and UDP, can be used with it. 

  • The whole packet provided to the IP layer, including headers from higher layers, is protected by IPsec 

  • It is more difficult to analyse communications because higher layer headers that carry port numbers are concealed. 

  • IPsec only functions from one network entity to another, not from one application process to another. Security can be implemented as a result without requiring modifications to individual user computers or applications. 

  • IPsec is most frequently used to create Virtual Private Networks (VPNs), either between two sites (gateway-to-gateway) or between a remote user and a business network (host-to-gateway). 

 

A collection of protocols called IPsec is used to secure network A collection of protocols called IPsec is used to secure network communications. It is a somewhat complicated mechanism because, rather than providing a clear definition of a particular encryption algorithm and authentication function, it offers a framework that permits the implementation of any encryption or authentication method that both communicating ends can agree upon. The two basic communication protocols employed by IPsec are Authentication Header (AH) and Encapsulating Security Payload (ESP). ESP can authenticate and encrypt the data passed over the connection, whereas AH can just authenticate. Between two endpoints, Transport Mode offers a secure connection without modifying the IP header. The whole IP payload packet is encapsulated in tunnel mode. A fresh IP header is added. Due to its ability to create a secure virtual tunnel through an insecure Internet, the latter is utilised to create a typical VPN. The secret values used for encryption or hashing must be known by both communication endpoints. Manual keys call for the manual entering of the secret values on both sides, presumably transmitted by some out-of-band technique, and IKE (Internet Key Exchange) is a sophisticated system for carrying out this online.   

Through this layer we will try to provide confidentiality to the company by enabling the communication nodes to encrypt messages which will prevent eavesdropping by the third parties using VPN. By encrypting traffic before it enters the public Internet and logically separating it from other traffic, VPN technology allows an institution's inter-office traffic to be sent over the public Internet. In addition to this security at network layer will also provide origin authentication and data integrity. It will guarantee that a packet received was genuinely communicated by the party acknowledged as the source in the packet header and will verify that the packet has not been tampered with in any way. It will also provide protection against the security attacks such as replay attacks.     

6)   Perimeter Layer: An additional layer of security between your assets and the internet is offered by a perimeter network, which might be a physical or logical network segment. Only intended traffic can enter your virtual network thanks to specialised network access control devices at the perimeter of a perimeter network. The advantage of perimeter networks is that you can concentrate your network access control management, monitoring, logging, and reporting on the equipment at the edge of your Azure virtual network. It is customary to implement distributed denial of service (DDoS) mitigation, intrusion detection/intrusion prevention systems (IDS/IPS), firewall rules and policies, web filtering, network antimalware, and other security measures on a perimeter network. Between your Azure virtual network and the internet, there is a point of contact for the network security devices. Depending on the location, this layer may also comprise security guards, fencing, security feeds, intrusion detection equipment, and other security measures. 

We have control over our network, technology, and data up to the perimeter. It is the edge of what we can influence and have a duty to protect. As a result, it serves as the initial line of defence against numerous outside threats to the network. Our network perimeter can be defined and secured in a number of ways. The precise methods for defending a network, however, are outside the purview of this essay because every network is unique. At the risk of oversimplifying, the steps and resources listed below will give a better idea of how to protect the network edge.  

  • Identify the perimeter – document all devices that have access to your network 

  • Learn what data flows through the perimeter 

  • Secure the devices that form the perimeter 

  • Guard the data that flows through the perimeter
